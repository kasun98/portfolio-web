Neuronbit

Predicting future prices in the volatile world of cryptocurrency has always been a challenging endeavor. Traditional models frequently rely on LSTM and RNN algorithms, employing techniques such as Z-score normalization and Min-Max normalization. However, these methods often fall short as they fail to adapt to the market’s breaking of previous minimums or maximums. Instead of predicting exact prices, I have developed a different approach through my project, NeuronBit, which aims to forecast the daily market direction of Bitcoin.

Github

The Concept

NeuronBit leverages a Deep neural network (DNN) with leaky ReLU activation and an intricate 14-layer architecture. Unlike conventional methods, NeuronBit predicts whether the next day's opening price will be higher or lower than today’s opening price. This directional prediction model provides traders with valuable insights into market trends, helping them make informed decisions.

Model Development

Feature engineering is the cornerstone of building an effective prediction model. For NeuronBit, I meticulously selected 26 features that significantly influence Bitcoin prices. These features were chosen based on their potential impact on market movements and their relevance to the prediction task. Notably, I chose not to apply normalization techniques, allowing the model to operate on raw data and adapt to market shifts more naturally.

Performance

NeuronBit has demonstrated exceptional performance, achieving a 97% F1 score on the test dataset. This high level of accuracy underscores the effectiveness of the feature selection and the robustness of the DNN architecture. Users can rely on NeuronBit to provide accurate market direction predictions, giving them a strategic edge in their trading activities.

ML Trading

In this project, I analyzed the prices of Deriv broker's synthetic trading indices, specifically focusing on Boom 500, Boom 1000, Crash 500, and Crash 1000 prices. The dataset consisted of 60 features extracted from 10 synthetic charts on MetaTrader 5. I experimented with different data transformations and machine learning models to predict these target prices.

Github

Feature Engineering

I started by extracting the price data for 10 synthetic charts from MetaTrader 5, resulting in a dataframe with 60 features. These features were analyzed for their correlation with the target synthetic indices: Boom 500, Boom 1000, Crash 500, and Crash 1000 prices.

Machine Learning Experiments

To evaluate the performance of different models and data transformations, I conducted the following experiments:

  • Raw Data: Used the original price data without any scaling.
  • Log Transformation: Applied natural logarithm to the price data.
  • Sine Transformation: Converted the price data to sine values.
  • Standard Scaling: Applied standard scaling to the price data.

    • For each experiment, I used XGBoost and k-Nearest Neighbors (KNN) models to predict the 'Is current market price good for a trade' (Binary classification). The performance was measured using accuracy and ROC-AUC scores.

    Observations

    • For Boom500 index analysis, both XGBoost and KNN algoriths performed well with log price data
    • For Boom1000 index analysis, XGBoost and KNN algoriths performed well with Standard scaled data and log price data respectively
    • For Crash500 index analysis, XGBoost and KNN algoriths performed well with Raw data and log price data respectively
    • For Crash1000 index analysis, XGBoost and KNN algoriths performed well with Standard scaled data and log price data respectively

    Results and Insights

    The comparison charts above summarizes the performance of XGBoost and KNN models across different data transformations. Here are some key insights:

    • Raw Data: XGBoost and KNN models performed reasonably well with raw data.
    • Log Transformation: Applying natural logarithm improved the performance of KNN significantly.
    • Sine Transformation: Sine transformation resulted in lower performance compared to other transformations.
    • Standard Scaling: Standard scaling provided a balanced performance for both models.

    Face Detection

    Face detection is a critical technology in various applications ranging from security systems to personal devices. Utilizing the power of deep learning, I developed a robust face detection model using VGG16, TensorFlow, OpenCV, and several other tools to achieve high accuracy and real-time performance.

    Github

    The face detection model leverages the VGG16 network, which is well-known for its excellent performance in image recognition tasks. By fine-tuning this pre-trained network, I adapted it to detect faces in real-time. The model not only identifies the presence of faces but also provides precise bounding boxes around detected faces, making it suitable for various practical applications.

    Data Collection and Annotation

    Data collection is a crucial part of developing a face detection model. Using OpenCV, I captured images from a webcam and stored them in a structured format. To annotate these images, I employed LabelMe, a powerful tool that allows for precise labeling of facial features and bounding boxes. This annotated dataset served as the foundation for training the model.

    Data Augmentation

    To enhance the model's robustness, I used Albumentations, a popular library for data augmentation. This step involved applying random transformations such as cropping, flipping, and adjusting brightness and contrast to create a diverse set of training images. This process ensures that the model can generalize well to various lighting conditions and orientations.

    Model Training

    The model was trained using a combination of classification and regression losses to accurately identify faces and predict their bounding boxes. The Adam optimizer with a carefully tuned learning rate ensured efficient convergence. The training process involved monitoring various metrics to achieve optimal performance.

    Technology Stack

    • OpenCV: For image capture and processing.
    • LabelMe: For annotating the dataset.
    • Albumentations: For data augmentation.
    • TensorFlow and Keras: For building and training the VGG16-based custom model.
    • Matplotlib: For visualizing data and model predictions.

    Falcon 9

    Reusing the first stage of Falcon 9 rockets significantly reduces the cost of space missions. While SpaceX advertises a launch cost of $62 million, other providers charge upwards of $165 million. The ability to predict successful landings of Falcon 9's first stage can provide invaluable insights for alternative companies competing against SpaceX. This project aims to develop a predictive model for the successful landing of Falcon 9's first stage, leveraging various data science methodologies and machine learning techniques.

    Github

    The Concept

    The project centers around building a robust machine learning model that predicts whether the first stage of Falcon 9 will land successfully. This model helps in estimating the potential cost savings of a launch, thereby aiding competitors in bidding processes.

    • Data Collection and Preparation: Data was gathered using a SpaceX REST API and web scraping techniques to compile detailed records of Falcon 9 launches. The data was then converted into a Pandas DataFrame and underwent extensive wrangling to clean and prepare it for analysis.
    • Exploratory Data Analysis: Initial analysis involved creating scatter plots and bar charts to uncover patterns and insights. SQL queries were also executed to further manipulate and sort the data. This step was crucial in understanding the underlying factors that influence successful landings.

    • interactive Visualization: An interactive dashboard was developed using Plotly Dash, featuring pie charts and scatter plots to provide dynamic data analysis capabilities. Additionally, Folium was used to create an interactive map that visualized launch site proximities and calculated distances.
    • Machine Learning Model Development: The core of the project involved training various classification models, including Support Vector Machines (SVM), Classification Trees, and Logistic Regression. Data was split into training and testing sets, and a hyperparameter grid search was conducted to optimize the models. The best-performing model was selected based on its performance on the test data.
